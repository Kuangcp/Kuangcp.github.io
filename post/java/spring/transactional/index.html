事务 和 AOP 同时使用 顺序问题

Afterreturning 和 Transactional

失效场景

  1. 引擎不支持事务，例如 MyISAM
  2. 注解所在类没有进入IOC容器
  3. 注解所在方法没有public修饰， 从反射和代理角度是可以实现的，只是规范上应该要public
  4. 该方法在当前类被调用，AOP没有生效
  5. 数据源没有加载事务管理器
  6. propagation 设置不正确
  7. 异常被catch或者rollback指定的类型不对应
